From Chatbots to Workers

An agent is a logic loop (code) that wraps around an LLM (the brain) to give it "hands."

Agent = LLM + Scaffolding (Code) + Tools

The Brain (LLM)

Makes decisions and understands the goal.

Planning

Breaks a big mission into a checklist of small steps.

Tools

Functions the code lets the LLM call (File System, API, Web Search).

Memory

Keeps track of what it already tried so it doesn't get stuck.

Modern Examples

Claude Code / Aider — Terminal agents that edit files and run tests

OpenAI Operator — Browser agent that books flights, fills forms

Devin — Fully autonomous software engineer agent

OpenClaw — Modular framework for custom agent stacks

The "Vibe" Check

Chatbot: A smart book. You ask, it answers.

Agent: A digital intern. You give a mission, it does the work and reports back.

Step 02

Chat Loop — Memory Through History

Add a loop and message history so the LLM remembers the conversation. This is how chatbots like ChatGPT work.

step02_chat_loop.py 
messages = []

while True:
    user_input = input("You: ")
    messages.append(user_input)

    response = llm.generate(messages)
    messages.append(response)
    print("Agent:", response.text)

Each turn: append user → call LLM → append reply. History grows, context deepens.

Step 03

Tool Use — Giving the Agent Hands

We give the LLM tools — functions it can call. The model decides when to use them. Press ▶ to step through the data flow →

step03_tool_use.py 
def run_bash(command): ...
tools = ["run_bash"]
system_message = {"role": "system", "content": "You have access to: run_bash"}
user_message = {"role": "user", "content": user_input}
messages = [system_message, user_message]
response = llm.generate(messages, tools=tools)
if response.tool_calls:
    result = run_bash(...)
messages.append({role:"tool", content: result})
response = llm.generate(messages)
print("Agent:", response.text)

Step 04

Agent Loop — Autonomy Through Nested Loops

The core pattern. Outer loop for human turns, inner loop for agent autonomy.

step04_agent_loop.py 
while True:    # Outer Loop: Conversation with Human
    user_input = input("You: ")
    messages.append(user_input)

    while True:  # Inner Loop: Agent autonomy
        response = llm.generate(messages, tools=[search_web])
        messages.append(response)

        # No tool calls → agent is done, respond to human
        if not response.tool_calls:
            print("Agent:", response.text)
            break        # Exit inner loop → back to outer

        # Has tool calls → execute all, then loop back
        for tool_call in response.tool_calls:
            result = execute(tool_call)
            messages.append(result)
        continue            # Loop back so LLM sees all results

The agent loops on its tool results until it has no more calls — then responds to the human.

Planning + Memory — Stateful Agents

Add todo tools for planning and file-based memory for persistence across sessions.

step05_planning.py + step06_memory.py 
state = {"todos": [], "memory": ""}

while True: # Inner Agent Loop
    context = messages + [f"State: {state}"]
    response = llm.generate(context, tools=[add_todo])

    if response.tool_calls:
        result = execute(response.tool_calls)
        state = update_state(state, result)
        continue

Model sees its own todo state and updates progressively. Memory survives program restart.

The Production Gap

The Reality Check

Our Python loops taught us the core. But production demands more than a while True: loop. Here's how modern systems solve for reliability, state, and safety.

Pillar 1 — Durable State

Tool: Claude Code

Secret: Auto Dream

Problem: Context "bloat" & memory decay in long sessions.

Fix: Background Defragmentation — periodically prunes & synthesizes logs into "working memory" while agent is idle.

Pillar 2 — Proactive Autonomy

Tool: OpenClaw

Secret: Local-First Heartbeat

Problem: Agents that only act when prompted — purely reactive.

Fix: Agent runs as a system service. Wakes up on cron triggers or events — no user prompt needed.

Pillar 3 — Secure Orchestration

Tool: OpenHands

Secret: Event-Stream Architecture

Problem: "Agent gone rogue" or catastrophic tool failures.

Fix: Docker Sandboxing — every execution in isolated, ephemeral containers. Every action = immutable event log.

The Architectural Level-Up

Scripts → Services

In production, we don't "run a script". We deploy an Agentic Daemon — always on, event-driven.

State Machines > Loops

If an agent fails at Step 4, it resumes from the last checkpointed state, not restart from zero.

Orchestrator-Critic

A "Senior Engineer" agent reviews the "Junior" agent's work before any code is committed or data is moved.

Final Thoughts

No matter how advanced the infrastructure — at its core, an agent is still just an LLM embedded in a while-loop, calling functions.

Don't be intimidated by the jargon. Once you build it yourself from these 10 steps, the magic is gone — and the actual engineering begins.

The while-loop is the 10% that makes it work. The engineering is the 90% that makes it production-ready.

LLM + Loop + Tools + Memory + Orchestration = Agent

We've spent this session looking under the hood. The "magic" is just a loop. The real work—the work that matters for GovTech—is the engineering we build around that loop to make it reliable, auditable, and safe. Let's get to work.
