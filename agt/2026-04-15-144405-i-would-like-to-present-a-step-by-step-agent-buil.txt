     Plan: Step-by-Step Agent Building Workshop

     Context

     Build a series of progressive, runnable Python files for a team
     presentation on agent building. Each file is self-contained, simple
     (~50-150 lines), and demonstrates one core concept. Uses MiniMax API
     (OpenAI-compatible) instead of Anthropic, making it vendor-agnostic and
     easy to swap to any OpenAI-compatible provider.

     API Setup: MiniMax OpenAI-compatible endpoint at https://api.minimax.io/v1,
      model MiniMax-M2.5, using the openai Python SDK.

     Security: API key stored in env var MINIMAX_API_KEY, never hardcoded.

     ---
     File Structure

     /home/yangzichao/agt/
       .env                    # MINIMAX_API_KEY=sk-cp-...
       step01_hello_llm.py     # Basic LLM call
       step02_chat_loop.py     # Interactive chat loop
       step03_tool_use.py      # Single tool (bash)
       step04_agent_loop.py    # Full agent loop with multiple tools
       step05_planning.py      # Agent with todo/planning
       step06_memory.py        # Agent with persistent memory (file-based)
       step07_subagent.py      # Spawning child agents
       step08_team.py          # Multi-agent team coordination

     ---
     Step-by-Step Breakdown

     Step 1: step01_hello_llm.py — "Hello, LLM"

     Slide concept: Connect to an LLM and get a response.
     What it does: Sends one message, prints the response.
     Key code (~20 lines):
     from openai import OpenAI
     client = OpenAI(api_key=os.environ["MINIMAX_API_KEY"],
     base_url="https://api.minimax.io/v1")
     response = client.chat.completions.create(
         model="MiniMax-M2.5",
         messages=[{"role": "user", "content": "Explain what an AI agent is in 2
      sentences."}]
     )
     print(response.choices[0].message.content)

     ---
     Step 2: step02_chat_loop.py — "The Conversation Loop"

     Slide concept: A while loop turns a single call into a chatbot.
     What's new: while True loop, message history accumulation.
     Key pseudocode:
     messages = [system_prompt]
     while True:
         user_input = input("> ")
         messages.append(user)
         response = llm(messages)
         messages.append(assistant)
         print(response)

     ---
     Step 3: step03_tool_use.py — "Giving the LLM Hands"

     Slide concept: Define a tool (run shell commands), let the LLM call it.
     What's new: Tool definitions, tool_calls detection, executing tools,
     returning results.
     Key pseudocode:
     tools = [{"type": "function", "function": {"name": "run_bash", ...}}]
     response = llm(messages, tools=tools)
     if response has tool_calls:
         result = execute(tool_call)
         messages.append(tool_result)
         response = llm(messages)  # LLM sees the result
     Demo: Ask "what files are in the current directory?" — agent runs ls.

     ---
     Step 4: step04_agent_loop.py — "The Agent Loop"

     Slide concept: Combine the chat loop + tool use into a full agent loop. The
      agent keeps calling tools until it decides to stop.
     What's new: Loop continues while stop_reason == "tool_calls". Multiple
     tools: run_bash, read_file, write_file.
     Key pseudocode:
     while True:
         response = llm(messages, tools)
         if response.has_tool_calls:
             for tool_call in response.tool_calls:
                 result = dispatch(tool_call)
                 messages.append(result)
         else:
             print(response.content)
             break  # or wait for next user input
     Demo: "Create a file called hello.txt with a poem, then read it back to
     me."

     ---
     Step 5: step05_planning.py — "An Agent That Plans"

     Slide concept: Add a todo/task tool so the agent plans before acting.
     What's new: todo_add, todo_update, todo_list tools backed by an in-memory
     list. System prompt instructs agent to plan first.
     Key insight: The agent becomes more reliable when it breaks tasks into
     steps and tracks progress.
     Demo: "Set up a Python project with a calculator module and tests."

     ---
     Step 6: step06_memory.py — "An Agent That Remembers"

     Slide concept: File-based memory that persists across sessions.
     What's new: save_memory and recall_memory tools that write/read from
     .agent_memory/ directory. Agent can store learnings and preferences.
     Key insight: Memory = files. Simple but powerful.
     Demo: Run once: "Remember that I prefer pytest over unittest." Run again:
     "What testing framework do I prefer?"

     ---
     Step 7: step07_subagent.py — "Divide and Conquer"

     Slide concept: Spawn a child agent with a focused task and clean context.
     What's new: spawn_subagent tool that creates a new agent loop with fresh
     messages but limited tools. Returns only a text summary to parent.
     Key insight: Context isolation prevents the parent's context from bloating.
     Demo: "Research what the top 3 Python web frameworks are, then write a
     comparison file."

     ---
     Step 8: step08_team.py — "Agent Teams"

     Slide concept: Multiple agents with different roles communicating via
     message passing.
     What's new: Multiple agent threads with inbox files. send_message,
     read_inbox tools. A "lead" agent coordinates "worker" agents.
     Key insight: Agents coordinate through simple file-based message queues.
     Demo: "Lead: plan a simple web app. Assign the backend to one agent and
     frontend to another."
